# 执行计划 (Executing Plans)

## 概述

加载计划、批判性审查、分批执行任务、批次之间报告评审进度。

**核心原则：** 分批执行，设置检查点供架构师评审。

**开始时声明：** "我正在使用 executing-plans skill 来执行这个计划。"

## 流程

### 步骤 1：加载并审查计划

1. 读取计划文件
2. 批判性审查 - 识别计划中的任何问题或疑虑
3. 如果有问题：在开始前向人类伙伴提出
4. 如果没有问题：创建 TodoWrite 并继续

### 步骤 2：执行批次

**默认：前 3 个任务**

对于每个任务：
1. 标记为 in_progress（进行中）
2. 严格按照每个步骤执行（计划包含 bite-sized 步骤）
3. 按规范运行验证
4. 标记为 completed（已完成）

### 步骤 3：报告

批次完成时：
- 展示已实现的内容
- 展示验证输出
- 说："准备接收反馈。"

### 步骤 4：继续

根据反馈：
- 如需要则应用更改
- 执行下一批次
- 重复直到完成

### 步骤 5：完成开发

所有任务完成并验证后：
- 声明："我正在使用 finishing-a-development-branch skill 来完成这项工作。"
- **必需的子技能：** 使用 superpowers:finishing-a-development-branch
- 遵循该技能进行测试验证、展示选项、执行选择

## 何时停止并寻求帮助

**立即停止执行，当：**
- 在批次中途遇到阻塞（缺少依赖、测试失败、指令不明确）
- 计划存在关键缺陷导致无法开始
- 你不理解某个指令
- 验证反复失败

**请求澄清而不是猜测。**

## 何时重新审视之前的步骤

**返回审查（步骤 1），当：**
- 伙伴根据你的反馈更新了计划
- 根本方法需要重新思考

**不要强行通过阻塞** - 停下来询问。

## 记住

- 首先批判性审查计划
- 严格按照计划步骤执行
- 不要跳过验证
- 计划提到时参考相关技能
- 批次之间：只报告并等待
- 遇到阻塞时停止，不要猜测
- 未经用户明确同意，绝不在 main/master 分支上开始实现

## 集成

**必需的工作流技能：**
- **superpowers:using-git-worktrees** - 必需：在开始前设置隔离的工作空间
- **superpowers:writing-plans** - 创建本技能要执行的计划
- **superpowers:finishing-a-development-branch** - 所有任务完成后完成开发
