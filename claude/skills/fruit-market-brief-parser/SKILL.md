# Fruit Market Brief Parser

## 目标

将水果行情简报解析为“单品行情级”结构化数据，服务于飞书表格写入和人工校验，重点保留趋势、价格、状态、品质、原因、建议和原文证据。

## 输入

输入通常是一段人工编写的水果行情文本，可能包含：

- 简报日期，如“7月6日”“今日”“周三”
- 区域或市场，如“华中”“华东”“南宁”“江南水果批发市场”
- 品类段落，如“芒果类”“瓜类”“蓝莓”
- 单品行情，如价格、涨跌、产地、上下架、缺货、品质、建议

可选输入：

- 基准日期：用于解释“今日、明天、后天、周三”等相对日期
- 写入目标：飞书表格或多维表格链接、工作表或数据表名称、字段映射

## 输出

默认输出 JSON 数组，一条 JSON 代表一个单品在一次行情简报中的一条行情记录。

字段固定如下：

```json
{
  "report_date_text": "原始日期文本",
  "region_market": "区域或市场",
  "category_name": "品类",
  "product_name": "单品",
  "origin_text": "产地",
  "trend": "行情趋势",
  "price_text": "价格信息原文",
  "status_text": "上架/下架/尾期/缺货/到货等状态",
  "quality_text": "品质/口感/硬度/糖度/损耗",
  "reason_text": "行情原因",
  "suggestion_text": "经营建议",
  "raw_evidence": "原文证据",
  "parse_note": "解析备注或不确定点"
}
```

飞书表格写入时，字段名默认映射为：

`简报日期、区域/市场、品类、单品、产地、行情趋势、价格信息、状态、品质/口感、原因、建议、原文证据、解析备注`

## 约束

1. 数据粒度固定为“单品行情级”：一行表示一个单品在一次简报中的一条行情记录；不按整份简报或纯品类存行。
2. 同一单品多个规格价格默认合并到一行；同一单品不同产地的趋势、状态或品质明显不同，可以拆多行。
3. 只抽取原文明确表达的信息；不得编造日期、价格、产地、原因或建议，相对日期默认保留原文。
4. 每行必须保留最小可追溯 `raw_evidence`；不确定内容写入 `parse_note`。
5. 非行情类内容直接跳过，不生成记录，例如问候、通知、@所有人、纯闲聊等。

## 执行流程

1. 识别简报元信息：
   - 提取日期到 `report_date_text`
   - 提取区域或市场到 `region_market`

2. 切分品类段落：
   - 识别“芒果类、瓜类、蓝莓、柠檬类”等标题
   - 将标题中的整体趋势作为该段默认趋势

3. 过滤非行情内容：
   - 仅处理包含单品、品类、价格、趋势、产地、品质、上下架、缺货、建议等行情要素的文本
   - 问候、通知、@所有人、纯闲聊、无商品行情含义的内容直接跳过
   - 跳过内容不需要输出记录

4. 拆分单品：
   - 每个具体水果单品生成一条记录
   - 多个单品并列时必须拆开
   - 多规格价格但同一行情判断时不要拆开

5. 抽取字段：
   - 产地写入 `origin_text`
   - 涨价、掉价、平稳、缺货等写入 `trend`
   - 价格、价差、百分比写入 `price_text`
   - 上架、下架、尾期、缺货、到货写入 `status_text`
   - 口感、硬度、糖度、损耗、坏果写入 `quality_text`
   - 下雨、货少、断档、到柜少写入 `reason_text`
   - 主推、囤货、不建议囤、少量多次写入 `suggestion_text`

6. 趋势归一：
   - “大涨价、持续上涨、上涨趋势”归一为“上涨”
   - “掉价、下调、下降趋势、略有降低”归一为“下跌”
   - “平稳、稳定、波动不大”归一为“稳定”
   - 同一行存在多种趋势时写“混合”，并在 `parse_note` 说明

7. 生成原文证据：
   - `raw_evidence` 使用支持该行判断的最小原文片段
   - 不要整篇简报塞入每一行

8. 输出前校验：
   - 每行必须有 `product_name`
   - 每行必须有 `raw_evidence`
   - 检查是否误合并多个单品
   - 检查是否过度拆分规格价格
   - 检查是否输出了原文不存在的信息

## 飞书表格写入流程

当用户要求写入飞书表格或多维表格时，按以下流程执行：

1. 确认写入目标：
   - 目标飞书表格或多维表格链接
   - 目标工作表或数据表名称
   - 是否追加写入，默认追加
   - 字段映射是否使用默认字段

2. 检查字段映射：
   - `report_date_text` -> `简报日期`
   - `region_market` -> `区域/市场`
   - `category_name` -> `品类`
   - `product_name` -> `单品`
   - `origin_text` -> `产地`
   - `trend` -> `行情趋势`
   - `price_text` -> `价格信息`
   - `status_text` -> `状态`
   - `quality_text` -> `品质/口感`
   - `reason_text` -> `原因`
   - `suggestion_text` -> `建议`
   - `raw_evidence` -> `原文证据`
   - `parse_note` -> `解析备注`

3. 写入前校验：
   - 空数组不写入
   - 缺少 `product_name` 或 `raw_evidence` 的记录不写入
   - 字段不存在时先提示用户，不擅自创建或改表结构

4. 执行写入：
   - 将 JSON 数组转换为飞书表格行
   - 默认追加到目标表末尾
   - 写入后返回成功条数、跳过条数和失败原因

5. 异常处理：
   - 无目标表链接时，只输出 JSON，不上传
   - 无权限、字段缺失、网络失败时，保留解析结果并说明失败原因
   - 不重复写入同一批数据，除非用户明确要求重试
