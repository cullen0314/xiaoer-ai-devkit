# Understand First

## 目标

在进入分析、方案、执行、写代码或生成文档前，先确认 AI 与用户对目标、背景、约束和下一步动作的理解一致，降低误解和返工。

## 适用场景

- 用户输入较长、较模糊或包含多个目标。
- 用户提出需求分析、方案设计、代码修改、问题排查、多步骤任务。
- 任务存在明显成本、风险或不可逆操作。
- 用户明确要求先理解、先确认、先对齐、复述一下。

## 不适用场景

- 简单事实问答、命令查询、翻译、格式转换等低风险任务。
- 用户已经明确要求直接执行，且目标、范围和操作方式都清楚。
- 已有更具体的技能更适合当前任务，例如想法梳理使用 `idea-clarifier`，苏格拉底式引导使用 `socratic-guide`。

## 工作流程

1. 阅读用户输入，提炼真实目标。
2. 用自己的话复述理解，不机械重复原文。
3. 明确列出目标、背景、约束、隐含意图和不确定点。
4. 如果任务涉及代码修改、文件修改、外部操作、数据变更、发布、推送或其他高影响动作，必须等待用户确认后再继续。
5. 如果用户纠正理解，先更新理解，再重新确认。
6. 用户确认后，再进入后续分析、方案或执行。

## 输出模板

```text
我的理解是：

1. 你想达成……
2. 当前背景是……
3. 关键约束是……
4. 我判断真正需要解决的是……
5. 还需要确认的是……

如果以上理解正确，我再继续下一步。
```

## 交互原则

- 简洁复述，避免长篇解释。
- 优先暴露不确定点，不假装已经理解。
- 不在确认前执行有影响的操作。
- 用户确认后，后续动作保持在已确认范围内。
