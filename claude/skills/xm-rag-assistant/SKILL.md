# xm-rag-assistant

当用户明确输入 `/xm-rag-assistant` 或 `xm-rag-assistant` 并提出问题时，使用本 Skill。不要因为普通代码问题自动触发本 Skill。

本 Skill 通过 `xm-rag` MCP 工具检索本地已索引代码和文档，帮助用户快速定位实现、理解业务链路、分析字段来源、追踪状态流转、评估影响范围。

## 输入格式

支持以下形式：

```text
/xm-rag-assistant mission operator 是在哪里赋值的
```

```text
/xm-rag-assistant
问题：查 mission state 流转
```

```text
/xm-rag-assistant
问题：订单状态变更影响哪些系统
项目：全部
```

## 两阶段流程

### 阶段一：列项目并等待确认

触发后先调用 `xm_rag_status` 获取当前已索引 collection，然后整理出：

- 当前可用代码项目
- 当前可用文档集
- 根据用户输入推断的建议项目范围

除非用户在同一条消息中已经明确写了“项目已确认”、“确认使用 xxx”或“确认全部项目”，否则不要直接检索业务问题。先向用户展示项目选择并等待确认。

推荐回复格式：

```text
当前可用项目：
- order-center
- summerfarm-wms

当前可用文档集：
- xm-docs

我建议本次使用：
- summerfarm-wms

请确认本次检索范围：使用建议项目、指定其他项目，还是使用全部项目？
```

如果无法从问题推断项目，建议使用全部项目：

```text
我建议本次使用：全部已索引项目
```

### 阶段二：确认后执行检索

用户确认项目范围后，再根据原始问题调用合适的 `xm-rag` MCP 工具并回答。

确认表达包括：

- `确认`
- `使用建议项目`
- `用 summerfarm-wms`
- `项目：summerfarm-wms`
- `全部`
- `所有项目`
- `all`
- `all-projects`

如果用户只回复项目名，也视为确认该项目。

## 项目选择规则

在阶段一生成建议范围时，按以下顺序判断：

1. 如果用户写了 `项目：全部`、`项目：所有`、`所有项目`、`全部项目`、`all` 或 `all-projects`，建议使用所有已索引代码项目，但仍需用户确认，除非用户同时写明“确认”。
2. 如果用户明确写了一个或多个项目名，例如 `项目：summerfarm-wms` 或 `项目：order-center,summerfarm-wms`，建议只使用这些项目，但仍需用户确认，除非用户同时写明“确认”。
3. 如果用户问题文本中自然出现已索引项目名，例如 `summerfarm-wms 里 mission 怎么取消`，建议只使用该项目。
4. 如果用户没有指定项目，建议使用所有已索引代码项目。
5. 如果用户指定的项目不在 `xm_rag_status` 返回的已索引项目中，停止检索并提示可用项目，让用户重新指定。

## 文档选择规则

- 如果用户的问题包含“文档”、“PRD”、“规则”、“说明”、“流程文档”、“根据文档”等表达，建议启用文档检索。
- 如果用户明确要求“只查文档”，只使用文档工具。
- 默认文档集是 `xm-docs`。
- 如果问题需要同时结合代码和文档，使用 `xm_rag_query`，并设置 `include_code=true`、`include_docs=true`。

## 工具选择规则

- 阶段一固定先使用 `xm_rag_status`。
- 找代码位置、入口、类、方法、字段使用点：使用 `xm_rag_search`。
- 解释单个或多个代码实现问题：使用 `xm_rag_ask`。
- 跨代码和文档综合分析：使用 `xm_rag_query`。
- 只查文档位置：使用 `xm_rag_search_docs`。
- 只基于文档回答：使用 `xm_rag_ask_docs`。
- 只有用户明确要求“更新索引”、“重建索引”、“重新索引”时，才使用 `xm_rag_index` 或 `xm_rag_index_docs`。索引工具默认 dry-run；除非用户明确要求写入索引，不要设置 `dry_run=false`。

## 检索回答格式

确认项目并完成检索后，使用以下结构，保持简洁：

```text
本次使用项目：
- ...

文档集：
- 使用/未使用 ...

问题理解：
...

结论：
...

关键证据：
- path/to/file:lines：说明

补充说明：
...
```

如果没有找到足够证据，明确说明：

```text
当前索引中没有检索到足够证据。已检索范围：
- 项目：...
- 文档集：...
```

## 回答要求

- 项目未确认前，只列项目和建议范围，不执行业务检索。
- 项目确认后，先给结论，再展开证据。
- 引用检索结果中的文件路径、行号范围、collection 或文档来源。
- 不要把检索结果包装成绝对事实；如果只是基于片段推断，要写明“基于当前检索结果推断”。
- 如果问题涉及高风险代码修改或复杂链路，建议继续读取原文件确认。
