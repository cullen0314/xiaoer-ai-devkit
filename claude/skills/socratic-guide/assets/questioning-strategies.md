# 苏格拉底式提问策略

## 问题类型识别

根据用户输入的关键词，识别问题类型：

| 关键词 | 问题类型 |
|-------|---------|
| 选型、选择、用A还是B、框架比较 | 技术方案选型 |
| Bug、报错、不work、为什么失败 | 代码问题排查 |
| 架构、设计、怎么设计、模块划分 | 架构设计 |
| 是什么、怎么理解、学习、入门 | 概念学习 |
| 其他 | 通用问题推理 |

---

## 各类型提问清单

### 技术方案选型

```
1. 澄清需求
   - "这个方案的核心目标是什么？"
   - "你最关注的点是什么？性能？开发效率？"
   - "预期解决什么问题？"

2. 探寻约束
   - "有哪些限制条件？"
   - "团队的技术背景是什么？"
   - "时间和预算如何？"

3. 挖掘假设
   - "你为什么倾向这个方案？"
   - "基于什么经验或信息？"
   - "这个假设可靠吗？"

4. 权衡比较
   - "如果选A方案，会有什么问题？"
   - "B方案的缺点是什么？"
   - "在什么情况下A更好？什么情况下B更好？"

5. 验证思考
   - "你现在的选择，能解决最开始的目标吗？"
   - "如果有新的信息，你会改变主意吗？"
```

### 代码问题排查

```
1. 观察现象
   - "你观察到的具体现象是什么？"
   - "预期行为是什么？"
   - "有什么错误信息？"

2. 缩小范围
   - "这个问题什么时候出现的？"
   - "改动前后有什么区别？"
   - "能复现吗？步骤是什么？"

3. 探寻假设
   - "你觉得可能是什么原因？"
   - "为什么这样想？"
   - "有什么证据支持？"

4. 验证推理
   - "如果这个推测是对的，验证方法是什么？"
   - "你怎么确认找到了根因？"

5. 总结预防
   - "问题解决后，如何避免再次发生？"
   - "代码需要加什么保护？"
```

### 架构设计

```
1. 理解场景
   - "系统的核心功能是什么？"
   - "预期规模多大？"
   - "用户主要操作是什么？"

2. 识别约束
   - "有哪些非功能需求？"
   - "性能要求是什么？"
   - "可用性要求？"

3. 拆解模块
   - "系统可以分成哪些核心模块？"
   - "它们如何协作？"
   - "数据怎么流动？"

4. 压力测试
   - "如果流量翻10倍，会发生什么？"
   - "哪个模块先扛不住？"
   - "单点故障在哪里？"

5. 权衡取舍
   - "在成本、复杂度、性能之间，你愿意牺牲什么？"
   - "这个设计过度了吗？"
```

### 概念学习

```
1. 了解现状
   - "你对这个概念的当前理解是什么？"
   - "你听过或看过什么相关内容？"

2. 发现困惑
   - "最让你困惑的点是什么？"
   - "哪些地方不太明白？"

3. 联系实际
   - "这个概念能解决什么实际问题？"
   - "你能想到的应用场景？"

4. 验证理解
   - "你能用自己的话解释一遍吗？"
   - "如果给别人讲解，你会怎么说？"
```

### 通用问题推理

```
1. 澄清问题
   - "这个问题本质上是在问什么？"
   - "你能举个具体例子吗？"

2. 探寻前提
   - "结论基于什么前提？"
   - "这些前提可靠吗？"

3. 多角度思考
   - "有没有其他可能？"
   - "从反面看呢？"
   - "别人会怎么看？"

4. 推演后果
   - "如果这样做，会有什么后果？"
   - "长期会怎样？"
   - "最坏的情况是什么？"
```
