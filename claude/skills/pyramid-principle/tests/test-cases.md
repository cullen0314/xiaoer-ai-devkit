# 金字塔原理 Skill - 测试用例

## 测试场景

### 场景1：快速模式 - 简单邮件优化

**输入**：
```
主题：关于项目进度的汇报

王总，

最近我们在做一个新项目，主要是关于电商平台的改造。
目前已经完成了一些工作，前端页面基本做完了，
后端API也差不多了。数据库还在优化中。
另外测试团队也介入了，发现了一些bug，
我们在努力修复中。大概还需要两周时间
能完成第一版本。

另外，我们人手有点紧张，如果能再
招一个前端开发就好了。预算方面
目前还够用。

如果有问题可以随时联系我。

小李
```

**预期输出**：
- 触发快速模式（< 500字）
- 诊断：结论不明确、结构混乱
- 优化：结论先行 + 结构重组

---

### 场景2：深度模式 - 工作报告优化

**输入**：
```
2024年工作总结

过去一年，我们团队完成了多个重要项目。
首先是用户中心系统的重构，这个项目
持续了三个月，涉及到前后端的改造。
通过这次重构，系统性能提升了30%，
用户投诉减少了50%。这个项目的
成功离不开团队的协作。

其次是数据分析平台的搭建。
我们从零开始，设计了整个数据仓库，
然后开发了BI报表系统。现在运营
同学可以自助查询数据了，不需要
每次都找技术支持。

另外，我们还做了一些技术债的清理工作。
比如升级了Spring Boot版本，修复了
一些安全漏洞。代码质量也提升了，
测试覆盖率从30%提高到70%。

当然也有一些问题。比如团队人员流动，
有两个同事离职了。新人的培养
还需要加强。另外，项目管理方面
还有改进空间，有时候会出现
需求变更频繁的情况。

明年的话，我们计划继续完善
用户中心，增加更多功能。数据分析
平台也要推广到更多部门。技术方面，
会关注云原生的趋势，考虑
将系统迁移到云平台。

以上就是我的工作总结。
```

**预期输出**：
- 触发深度模式（≥ 500字）
- 提问：目标受众、沟通目标、使用场景
- 生成金字塔结构图
- 输出重组后的内容

### 场景3：技术方案优化

**输入**：
```
关于推荐系统的技术方案

我们公司需要做一个推荐系统。
目前用户反馈说找不到想要的商品，
搜索功能也不太好用。

技术选型方面，可以考虑用协同过滤，
也可以用内容推荐。协同过滤
比较成熟，但需要大量用户行为数据。
内容推荐可以解决冷启动问题。

数据方面，我们需要收集用户的
浏览、点击、购买行为。还要
建立用户画像和商品画像。

系统架构的话，可以用离线计算
加在线服务的方式。离线部分
用Spark做批量处理，在线部分
用Redis缓存推荐结果。

当然也可以考虑用深度学习，
比如神经网络推荐。不过这个
技术门槛比较高，团队可能
需要学习。

部署方面，可以用Docker容器化，
用Kubernetes管理。这样方便
扩展和容灾。

以上就是我的想法。
```

**预期输出**：
- 诊断：结构混乱，论据无序
- 建议：SCQA模型引入 + 方案对比结构
- 优化：问题背景 → 方案对比 → 技术选型 → 实施计划

---

### 场景4：MECE 违规检测

**输入**：
```
销售渠道分析

我们的销售渠道分为：
1. 线上渠道
2. 电商平台
3. 社交媒体
4. 线下门店
5. 代理商渠道
6. 直销渠道
```

**预期输出**：
- 诊断：MECE违规
- 问题：电商平台属于线上渠道（概念包含）
- 建议：重新分类为：电商平台、社交媒体、线下门店、代理商

### 场景5：演讲大纲优化

**输入**：
```
关于AI时代的个人发展

大家好，今天我想和大家聊聊AI时代
的个人发展。现在AI发展很快，
ChatGPT、Claude这些工具很强大。

我们需要学会使用这些工具。
可以提高工作效率。比如写代码、
写文档都可以用AI帮忙。

另外，我们也要保持学习能力。
技术在不断进步，我们也要
不断学习新东西。

还有就是软技能也很重要。
沟通能力、团队合作能力，
这些是AI替代不了的。

所以我们要积极拥抱AI，
同时也要发挥人类的独特优势。
让我们一起努力吧。

谢谢大家。
```

**预期输出**：
- 触发快速模式
- 诊断：结构松散，论据不清晰
- 优化：核心结论 → 三个支撑点（工具使用、持续学习、软技能）→ 行动号召

## 边界测试用例

### 边界1：超长内容

**输入**：5000字以上的文档

**预期输出**：
```
⚠️ 内容过长提示

检测到内容长度为 xxx 字，超过建议的 5000 字上限。

建议选择：
1. 分段处理 - 指定需要优化的部分
2. 摘要优化 - 只优化核心摘要部分
3. 继续处理 - 将处理全部内容（可能质量下降）
```

### 边界2：无结构内容

**输入**：完全不连贯的片段

**预期输出**：
```
⚠️ 难以识别结构

当前内容难以识别出清晰的逻辑结构。

建议：
1. 提供更多上下文信息
2. 说明内容的沟通目标
3. 进入深度模式进行交互式优化
```

### 边界3：代码+文档混合

**输入**：包含代码块的技术文档

**预期输出**：
- 保留代码块格式
- 只优化说明文字
- 提示：代码部分已保留原样

## 验收标准

### 功能验收

- [x] 快速模式正确触发（< 500字，结构清晰）
- [x] 深度模式正确触发（≥ 500字，或结构混乱）
- [x] 分析报告格式正确
- [x] 金字塔结构图正确生成
- [x] MECE违规正确识别
- [x] 优化后内容符合金字塔原理
- [x] 用户确认后输出最终版本

### 质量验收

- [x] 优化后内容逻辑清晰
- [x] 结论明确且在开头
- [x] 论据支撑结论
- [x] 分类符合MECE原则
- [x] 保持原意不变

### 体验验收

- [x] 提示信息友好
- [x] 问题逐步提问（深度模式）
- [x] 确认机制清晰
