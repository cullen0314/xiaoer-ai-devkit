# 金字塔原理 - 诊断规则

## 结构诊断规则

### 1. 结论先行检测

**规则**：前100字内应有明确的核心观点

**判断标准**：
```
✅ 有明确结论：
- 开头有"我认为"、"建议"、"总之"等标志性词语
- 第一段/句表达完整观点
- 能用一句话概括全文主旨

❌ 无明确结论：
- 开头是背景铺垫
- 开头是问题陈述
- 需要读到最后才知道作者观点
```

**示例**：
```
❌ 问题示例：
"随着市场竞争的加剧，我们公司面临着前所未有的挑战。
客户需求不断变化，技术更新速度加快，团队也需要适应..."

✅ 优化后：
"建议公司加快数字化转型，以应对市场竞争加剧的挑战。
随着客户需求变化和技术更新加快，当前传统模式已无法满足..."
```

---

### 2. 以上统下检测

**规则**：上层思想必须是对下层的总结或概括

**判断标准**：
```
✅ 符合以上统下：
- 标题能概括段落内容
- 论点能概括其下的论据
- 每一层级都能回答"为什么"

❌ 违反以上统下：
- 标题与内容不匹配
- 论据与论点无关
- 层级关系混乱
```

**示例**：
```
❌ 问题示例：
"提升用户体验
- 降低服务器成本
- 优化数据库查询"
（论据与论点无关）

✅ 优化后：
"提升用户体验
- 简化注册流程
- 优化页面加载速度
- 改进客服响应机制"
```

---

### 3. 归类分组检测

**规则**：同类内容应归为一组，每组不超过7个要点

**判断标准**：
```
✅ 归类合理：
- 同组内容有共同特征
- 每组有明确的分类标准
- 每组不超过7个要点（符合7±2法则）

❌ 归类混乱：
- 相关内容分散在不同位置
- 无关内容放在同一组
- 单组超过7个要点
```

**示例**：
```
❌ 问题示例（分散）：
"项目进展：
1. 完成前端开发
2. 确定了新的配色方案
3. 后端API已就绪
4. 修改了logo
5. 数据库优化完成
6. 更新了字体
7. 完成测试
8. 调整了布局"

✅ 优化后（归类）：
"项目进展：
【技术进展】
1. 前端开发完成
2. 后端API就绪
3. 数据库优化完成
4. 测试完成

【设计调整】
1. 确定配色方案
2. 修改logo
3. 更新字体
4. 调整布局"
```

---

### 4. 逻辑递进检测

**规则**：内容应按明确的逻辑顺序排列

**判断标准**：
```
✅ 逻辑顺序清晰：
- 时间顺序：步骤1→步骤2→步骤3
- 结构顺序：整体→局部，或外→内
- 重要性顺序：最重要→次重要→次要

❌ 逻辑顺序混乱：
- 前后跳跃，无规律可循
- 顺序颠倒（如结果在前原因在后）
- 同一层级混用多种顺序
```

## MECE 违规检测

### 检测规则

**MECE = Mutually Exclusive + Collectively Exhaustive**
- **相互独立**：各部分之间没有重叠
- **完全穷尽**：所有可能性都被覆盖

### 常见违规类型

#### 1. 时间重叠
```
❌ 错误：
- 儿童期：0-12岁
- 青少年期：10-18岁
- 成年期：18岁以上

✅ 正确：
- 儿童期：0-12岁
- 青少年期：13-18岁
- 成年期：19岁以上
```

#### 2. 概念包含
```
❌ 错误：
- 线上渠道
- 电商渠道
- 社交媒体渠道

（电商是线上的子集，违反相互独立）

✅ 正确：
- 电商平台
- 社交媒体
- 官方网站
```

#### 3. 层级混乱
```
❌ 错误（不同层级并列）：
- 市场营销
- SEO优化
- 内容营销
- 付费广告

（SEO和内容营销是市场营销的子集）

✅ 正确：
- 付费推广
  - 搜索广告
  - 社交广告
- 内容运营
  - SEO优化
  - 内容营销
```

#### 4. 遗漏可能性
```
❌ 不完整：
用户可以分为：
- 新用户
- 活跃用户

✅ 完整：
- 新用户
- 活跃用户
- 流失用户
（或：新用户、老用户，更简洁）
```

## 主题数量检测

### 检测方法

1. **关键词聚类**：提取高频词，分析主题聚集
2. **段落主题分析**：每段的核心主题是什么
3. **主题切换检测**：统计主题切换次数

### 判断标准
```
单主题：段落间主题一致性 > 80%
多主题：段落间主题一致性 < 60%，或主题切换 > 3次
```

## 模式选择规则

### 快速模式触发（满足任一）
```
IF (字数 < 500 AND 结构清晰) OR
   (单一主题) OR
   (用户显式指定 --fast)
THEN 快速模式
```

### 深度模式触发（满足任一）
```
IF (字数 >= 500) OR
   (多主题混合) OR
   (结构混乱检测 >= 2个问题) OR
   (用户显式指定 --deep)
THEN 深度模式
```

---

## 结构混乱评分

| 问题 | 扣分 |
|------|------|
| 无明确结论 | -3 |
| 论据不支撑结论 | -2 |
| 归类混乱 | -2 |
| 逻辑顺序混乱 | -2 |
| MECE违规 | -2 |
| 主题切换频繁 | -1 |

**评分标准**：
- 0分：结构清晰，快速模式
- -1~-2分：结构一般，可选深度模式
- -3分以下：结构混乱，深度模式

## 特殊情况处理规则

```
IF 内容长度 > 5000字 THEN
    提示："内容较长，建议分段处理或指定重点部分"

IF 完全无结构可识别 THEN
    提示："难以识别内容结构，建议提供更多上下文信息"

IF 检测到代码块 THEN
    保留代码块格式，仅优化说明文字

IF 多语言混合 THEN
    分语言检测并分别处理
```
