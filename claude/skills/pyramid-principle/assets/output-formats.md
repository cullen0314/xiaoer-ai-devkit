# 金字塔原理 - 输出格式定义

## 分析报告格式

### 标准格式

```markdown
## 📊 金字塔原理分析报告

### 内容诊断
- 📝 字数：xxx 字
- 🎯 主题数量：x 个
- 📊 结构评估：清晰 ✅ / 一般 ⚠️ / 混乱 ❌

### 主要问题
1. ❌ 结论不明确 - 开头没有核心观点
   - 位置：第一段
   - 建议：用一句话概括核心观点

2. ❌ 论据混杂 - 多个观点混在一起
   - 位置：第二段
   - 建议：按逻辑拆分独立论点

3. ⚠️ MECE问题 - 分类有重叠
   - 位置：第三部分
   - 建议：重新划分分类标准

### 优化建议
- 建议结构：[问题-原因-方案] / [目标-现状-计划]
- 核心结论：[一句话提炼]
- 论据框架：
  1. [论据一]
  2. [论据二]
  3. [论据三]

### 优化后的内容
---
[优化后的完整内容]
---

### 改动说明
- 将结论移至开头
- 重组论据逻辑顺序
- 合并相关内容
- 删除冗余信息
```

---

## 快速模式输出格式

```markdown
## 📊 金字塔原理分析（快速模式）

### 诊断结果
内容长度：xxx字 | 结构评估：[清晰/一般/混乱]

### 发现的问题
1. [问题一] - 简要说明
2. [问题二] - 简要说明

### 优化版本
---
[优化后的内容]
---

是否使用此版本？(确认/需要调整)
```

## 深度模式输出格式

### 阶段一：分析报告

```markdown
## 📊 金字塔原理深度分析

### 内容诊断
- 📝 字数：xxx 字
- 🎯 主题数量：x 个
- 📊 结构评估：清晰 ✅ / 一般 ⚠️ / 混乱 ❌

### 检测到的问题
[详细列出所有问题，与标准格式相同]
```

### 阶段二：上下文确认

```markdown
### 上下文信息
- 目标受众：[上级/同事/客户/公众]
- 沟通目标：[说服/汇报/说明/请求]
- 使用场景：[邮件/报告/演讲/会议]
```

### 阶段三：金字塔结构图

```markdown
### 金字塔结构

                    ┌─────────────────┐
                    │   核心结论      │
                    │ [一句话概括]    │
                    └────────┬────────┘
                             │
            ┌────────────────┼────────────────┐
            ▼                ▼                ▼
     ┌───────────┐    ┌───────────┐    ┌───────────┐
     │  论据一   │    │  论据二   │    │  论据三   │
     │ [论点]    │    │ [论点]    │    │ [论点]    │
     └─────┬─────┘    └─────┬─────┘    └─────┬─────┘
           │                │                │
     ┌─────┴─────┐    ┌─────┴─────┐    ┌─────┴─────┐
     ▼           ▼    ▼           ▼    ▼           ▼
  [事实A]     [事实B]  [事实C]    [事实D]  [事实E]    [事实F]
```

### 阶段四：优化后的内容

```markdown
### 优化后的完整内容
---
[根据金字塔结构重组的完整内容]
---

### 改动说明
1. 结论部分：
   - 提取核心观点前置
   - 使用SCQA模型引入

2. 论据部分：
   - 重组为三个支撑点
   - 按重要性排序

3. 细节部分：
   - 补充具体数据
   - 删除冗余信息

### 逻辑关系
- 结论 ← 论据一 ← 事实A、事实B
- 结论 ← 论据二 ← 事实C、事实D
- 结论 ← 论据三 ← 事实E、事实F
```

## 确认提示

### 快速模式确认

```markdown
---
✅ 优化完成

以上是基于金字塔原理的优化版本。

请问：
1. 是否满意此版本？（回复"确认"或"需要调整"）
2. 如需调整，请说明具体要求
```

### 深度模式确认

```markdown
---
✅ 深度优化完成

基于您提供的上下文信息：
- 目标受众：{audience}
- 沟通目标：{goal}
- 使用场景：{scenario}

已生成优化版本。请确认：
1. 结构是否符合预期？
2. 内容是否完整？
3. 风格是否恰当？

回复"确认"使用此版本，或提出调整建议。
```

## 错误处理输出

### 内容过长

```markdown
⚠️ 内容过长提示

检测到内容长度为 xxx 字，超过建议的 5000 字上限。

建议选择：
1. 分段处理 - 指定需要优化的部分
2. 摘要优化 - 只优化核心摘要部分
3. 继续处理 - 将处理全部内容（可能质量下降）

请选择处理方式：
```

### 无结构识别

```markdown
⚠️ 难以识别结构

当前内容难以识别出清晰的逻辑结构。

建议：
1. 提供更多上下文信息
2. 说明内容的沟通目标
3. 进入深度模式进行交互式优化

是否进入深度模式？(是/否)
```

### 用户拒绝建议

```markdown
❌ 优化建议未采纳

我已尝试基于金字塔原理进行优化，但似乎未满足您的需求。

您可以：
1. 说明具体的调整要求
2. 提供更多上下文信息
3. 选择其他优化方向

或直接使用原始内容。

请告诉我您的想法：
```

### SCQA 模型图

```
┌─────────────────────────────────────────┐
│           SCQA 序言结构                 │
├─────────────────────────────────────────┤
│                                         │
│  S - Situation (情境)                   │
│  ┌─────────────────────────────────┐   │
│  │ 当前稳定状态，公认的背景事实     │   │
│  └─────────────────────────────────┘   │
│                 │                       │
│                 ▼                       │
│  C - Complication (冲突)                │
│  ┌─────────────────────────────────┐   │
│  │ 发生的变化/问题，打破稳定状态     │   │
│  └─────────────────────────────────┘   │
│                 │                       │
│                 ▼                       │
│  Q - Question (疑问)                   │
│  ┌─────────────────────────────────┐   │
│  │ 基于S和C自然产生的问题           │   │
│  └─────────────────────────────────┘   │
│                 │                       │
│                 ▼                       │
│  A - Answer (答案)                     │
│  ┌─────────────────────────────────┐   │
│  │ 你的核心结论/方案                │   │
│  └─────────────────────────────────┘   │
│                                         │
└─────────────────────────────────────────┘
```
