# 金字塔原理优化 Skill

## 概述

应用芭芭拉·明托的金字塔原理，帮助用户重组和优化各类内容，使其结构清晰、逻辑严密、易于理解。

**核心原则**：先分析建议，由用户确认后再输出优化版本。

## 何时使用

当用户说以下类似话语时触发：
- "用金字塔原理优化这段话"
- "帮我重组这个文档"
- "这个内容结构不清晰，重新整理下"
- "pyramid [内容]"
- "/pyramid [内容]"

## 工作流程

```
内容接收 → 智能诊断 → 模式选择 → 优化处理 → 展示建议 → 用户确认 → 输出最终版本
```

### 快速模式触发条件（满足任一）
- 内容 < 500字
- 单一主题
- 结构清晰
- 用户指定 `--fast`

### 深度模式触发条件（满足任一）
- 内容 ≥ 500字
- 多主题混合
- 结构混乱
- 用户指定 `--deep`

## 快速模式流程

1. **结构诊断** - 分析内容问题
2. **生成建议** - 输出优化方案
3. **展示版本** - 给出优化后的内容
4. **等待确认** - 用户确认后最终输出

## 深度模式流程

1. **结构诊断** - 分析内容问题
2. **提问澄清** - 逐个询问上下文信息：
   - 目标受众是谁？
   - 沟通目标是什么？
   - 使用场景是什么？
3. **基于上下文重构** - 结合答案重新组织
4. **展示金字塔结构** - 可视化呈现结构
5. **等待确认** - 用户确认后生成最终内容

## 输出格式

### 分析报告

```markdown
## 📊 金字塔原理分析报告

### 内容诊断
- 字数：xxx
- 主题数量：x
- 结构评估：清晰/一般/混乱

### 主要问题
1. ❌ 结论不明确 - 开头没有核心观点
2. ❌ 论据混杂 - 多个观点混在一起
3. ⚠️ MECE问题 - 分类有重叠

### 优化建议
- 建议采用 [结构类型]
- 核心结论：[提取出的结论]
- 论据框架：[金字塔结构]

### 优化后的内容
[展示优化版本]
```

### 深度模式额外输出

```markdown
### 上下文信息
- 目标受众：xxx
- 沟通目标：xxx
- 使用场景：xxx

### 金字塔结构图
        核心结论
         /    \
      论据1   论据2
```

## 命令格式

```bash
# 直接传入内容
/pyramid [内容...]

# 引用文件
/pyramid --file document.md

# 指定模式
/pyramid --fast [内容]
/pyramid --deep [内容]

# 交互模式
/pyramid --interactive

# 组合使用
/pyramid --file report.md --deep
```

## 核心判断逻辑

### 结构混乱检测
- 前100字内没有明确结论 → 结论不明确
- 段落间逻辑连接词缺失/混乱 → 逻辑跳跃
- 同一层次内容超过7点 → 归类不足

### MECE 违规检测
- 存在时间重叠的分类
- 存在概念包含关系的并列项

### 主题数量检测
- 通过关键词聚类判断主题数量
- 段落主题切换 > 3次视为多主题

## 支持的内容类型

- 文本类：邮件、报告、文章、需求文档
- 技术类：代码、技术方案、API文档
- 演示类：PPT大纲、演讲稿
- 通用：任何需要清晰结构的内容

## 特殊情况处理

1. **内容过长（>5000字）**
   → 提示分段处理或只处理前N字

2. **内容完全无结构**
   → 引导用户提供更多上下文

3. **多语言混合**
   → 检测语言并分别处理

4. **代码/技术内容**
   → 保留代码块，优化说明文字

5. **用户拒绝所有建议**
   → 输出原始内容 + "如需其他帮助请告知"

## 交互式问题（深度模式）

进入深度模式时，按顺序提问：

1. **【内容确认】**
   "这是一段关于 [主题] 的内容，对吗？"

2. **【目标受众】**（多选）
   "这内容是给谁看的？
    A. 上级/领导
    B. 同事/团队
    C. 客户/外部
    D. 公众/通用"

3. **【沟通目标】**（多选）
   "你希望达到什么目的？
    A. 说服对方接受观点
    B. 汇报工作/进展
    C. 解释/说明某事
    D. 请求批准/资源
    E. 其他"

4. **【使用场景】**（单选）
   "这内容用在什么场景？
    A. 邮件/消息
    B. 报告/文档
    C. 演讲/演示
    D. 会议讨论"

5. **【紧急程度】**
   "是否需要快速优化？（是=快速模式，否=继续深度分析）"
