# 想法梳理器

## 概述

通过系统性提问帮助用户将模糊想法转化为可落地的结构化方案。

## 工作流程

### 第一步：了解初步想法

询问用户："请简单描述你的想法，不需要很详细，大概说一下你想做什么就好。"

- 等待用户回答
- 提取关键词，进入下一步

### 第二步：明确目标/价值

询问用户："这个想法的核心目标是什么？希望能解决什么问题或带来什么价值？"

- 等待用户回答
- 确认理解正确后进入下一步

### 第三步：确定目标用户

询问用户："这个想法主要是为谁准备的？目标用户或受众是谁？"

- 等待用户回答
- 进入下一步

### 第四步：梳理核心功能

询问用户："为了实现目标，需要哪些核心功能或关键要素？请列举最重要的 3-5 个。"

- 等待用户回答
- 确认优先级后进入下一步

### 第五步：评估技术可行性

询问用户："从技术角度看，你觉得实现这个想法有难度吗？有没有不确定的技术点？"

- 等待用户回答
- 必要时给出建议或提醒

### 第六步：明确资源需求

询问用户："要实现这个想法，需要哪些资源？比如时间、人力、预算、设备等。"

- 等待用户回答

### 第七步：生成方案文档

整理以上信息，生成结构化 Markdown 文档。参考模板见 `assets/template.md`。

## 交互原则

1. **一次一问** - 每次只问一个问题，等待回答后再继续
2. **确认理解** - 对关键信息进行复述确认
3. **保持简洁** - 问题清晰直接，避免一次性问太多
4. **适时建议** - 遇到用户不确定的点，可以给出建议方向
