### 阶段二：Entity 和 Repository

#### 任务 2.1：创建 Entity

**文件结构**：
```

  └── {package路径}/entity/
      └── {EntityName}.java  ← 新建
```

**代码要点**：
```java
@Entity
@Table(name = "{表名}")
@Data
public class {EntityName} {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    // 其他字段...

    @CreatedDate
    private LocalDateTime createdAt;

    @LastModifiedDate
    private LocalDateTime updatedAt;
}
```

**验证命令**：
```bash
./gradlew compileJava
# 预期：编译成功，无错误
```

### 阶段三：Service 层

#### 任务 3.1：创建 Service

**文件结构**：
```
src/main/java/
  └── {package路径}/service/
      ├── {ServiceName}.java       ← 新建（接口）
      └── impl/
          └── {ServiceName}Impl.java  ← 新建（实现）
```

**代码要点**：
```java
// 接口定义
public interface {ServiceName} {
    {ResponseDTO} {methodName}({RequestDTO} request);
}

// 实现类
@Service
@RequiredArgsConstructor
public class {ServiceName}Impl implements {ServiceName} {
    private final {RepositoryName} repository;
    private final {OtherService} otherService;

    @Transactional
    @Override
    public {ResponseDTO} {methodName}({RequestDTO} request) {
        // 实现逻辑
    }
}
```

**验证命令**：
```bash
./gradlew test --tests {ServiceName}Test
# 预期：测试通过
```

#### 任务 4.1：创建 Controller

**代码要点**：
```java
@RestController
@RequestMapping("/api/{resource}")
@RequiredArgsConstructor
public class {ControllerName} {
    private final {ServiceName} service;

    @PostMapping
    public ResponseEntity<{ResponseDTO}> create(@RequestBody @Valid {RequestDTO} request) {
        {ResponseDTO} response = service.{methodName}(request);
        return ResponseEntity.ok(response);
    }
}
```

**验证命令**：

## 6.4 新会话恢复指南

如果需要在新会话中继续开发，按以下步骤操作：

1. **确认文档位置**：
   ```bash

   ```

2. **读取技术设计文档**，重点查看：
